# 1차 검색 실험 결과 (E1 청킹 · E2 임베딩)

> 목적1(사내 규정 RAG 최적화 기법 시범)의 첫 정량 비교. 금융 법령 테스트베드.
> 결과는 사내 규정 RAG로 전이할 **기법 선택 근거**다. (방법론: [AXES.md](AXES.md))

## 셋업
- 코퍼스: 핵심 금융법 32개(13군, 조문 3,609 + 별표 126), 청크 ~3,251(별표 md 포함)
- 골드셋: **240문** (factoid 60[격식30+구어30쌍] / crossref 60 / byeolpyo 60 / multihop 60)
  - 생성기 Mistral Small 4(공식 reasoning_effort=none) + 일관성 round-trip 필터
- 메트릭: recall@k, MRR, nDCG@10 (유형별·register별 분해), gold = 조문/별표 uid

## 결과 1 — 검색기 × 청킹 (전체 recall@5)
| 청커 | BM25 | Vector(KURE-v1) |
|---|---|---|
| **article (조)** | **0.835** | **0.767** |
| fixed (고정토큰) | 0.767 | 0.696 |
| parent (small-to-big) | 0.750 | 0.754 |
| hang (항) | 0.725 | 0.738 |

- **조(article) 단위가 양쪽 최고** — 법령 인용·의미의 자연 단위.
- **fixed는 BM25엔 좋고(0.767) 벡터엔 나쁨(0.696)** — 큰 청크가 어휘매칭엔 유리, 의미초점은 희석.
- **parent-doc는 벡터 랭킹을 개선**(ndcg/mrr ↑, recall@5 0.754) — 작은단위 검색→큰단위 반환 효과.

## 결과 2 — 임베딩 모델 (article + vector, recall@5 / ndcg@10)
| 임베더 | recall@5 | ndcg@10 |
|---|---|---|
| **KURE-v1** | **0.767** | **0.672** |
| KoE5 | 0.710 | 0.646 |
| BGE-M3 | 0.702 | 0.610 |

→ **KURE-v1 우승** (한국어 특화 BGE-M3 파인튜닝). 예측 일치.

## 결과 3 — 검색기 비교 (article 고정): **BM25 ≳ 벡터**
| | 전체 | crossref | byeolpyo | factoid | multihop |
|---|---|---|---|---|---|
| **BM25** | **0.835** | **0.750** | **0.950** | 0.900 | 0.742 |
| Vector(KURE) | 0.767 | 0.483 | 0.883 | **1.000** | 0.700 |

- **법령 RAG는 BM25가 강력한 베이스라인** — 법령명·조문번호·수치 등 어휘 매칭이 많음.
- **crossref: BM25(0.75) ≫ 벡터(0.48)** — 「법령명」 참조는 어휘 매칭. 벡터의 명확한 약점.
- **factoid: 벡터(KURE 1.0) > BM25** — 의미 질문엔 dense가 우위.
- ⇒ **하이브리드(BM25+dense)가 유망** (상호보완적 강점).

## 결과 4 — 🔥 반직관적 발견
**구어체 질문이 격식체보다 검색이 쉬움** (모든 임베더 공통):
| 임베더 | formal recall@5 | colloquial recall@5 |
|---|---|---|
| KURE-v1 | 0.733 | **1.000** |
| KoE5 | 0.674 | 0.967 |
| BGE-M3 | 0.669 | 0.933 |

- HyPE 가정("구어 질문은 어휘격차로 불리")과 **반대**. 구어 질문이 더 짧고 직접적이라 의미 매칭이 쉬움.
- (BM25에선 formal 0.836 ≈ colloquial 0.833으로 격차 없음)
- ⇒ **HyPE 효과는 가정 말고 반드시 실측**(E5). 이 골드셋에선 어휘격차가 약함 — 더 거친 실사용자 질문이 필요할 수도.

**crossref가 전 구성 최난도** (벡터 0.48, BM25 0.75) — 그래프RAG(LightRAG)·하이브리드의 핵심 타깃.

## 사내 규정 RAG 전이 시사점
1. **조 단위(=문서 섹션 단위) 청킹 + BM25를 강력한 기본**으로 깔 것 (사내 규정도 조항·섹션 구조).
2. **하이브리드(BM25+dense)**가 단일 방식보다 나을 가능성 — 정의/의미 질문은 dense, 명칭/번호/표 조회는 BM25.
3. **한국어 임베딩은 KURE-v1**.
4. 교차참조(규정 간 인용)는 별도 처리 필요(그래프/하이브리드).
5. 합성 질문의 register가 실사용자와 다를 수 있음 — 골드셋 현실성 검증(인간 표본) 권장.
